# Video Testing Guide

This guide explains how to create and use test videos for the NicheStream upload and streaming functionality.

## Quick Start

### 1. Generate a Test Video

```bash
# Generate a 10-second test video (default)
node test/fixtures/generate-test-video.js

# Or specify custom output path and duration
node test/fixtures/generate-test-video.js ./my-test-video.mp4 15
```

**Prerequisites**: ffmpeg must be installed
```bash
# macOS
brew install ffmpeg

# Ubuntu / Debian
sudo apt-get install ffmpeg

# Windows
choco install ffmpeg
```

### 2. Upload Video via API

#### Option A: Using curl

```bash
# Step 1: Get upload URL (requires authentication)
curl -X POST http://localhost:8787/api/videos/upload-url \
  -H "Authorization: Bearer YOUR_AUTH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"maxDurationSeconds": 600}' \
  > upload-response.json

# Extract uploadUrl and videoId
UPLOAD_URL=$(jq -r '.uploadUrl' upload-response.json)
VIDEO_ID=$(jq -r '.videoId' upload-response.json)

# Step 2: Upload video to Cloudflare Stream
curl -X POST "$UPLOAD_URL" \
  -F "file=@test/fixtures/test-video-10s.mp4"

# Response includes streamVideoId for fetching playback data
```

#### Option B: Using the Web UI

1. Navigate to the upload form at `http://localhost:3000/upload`
2. Click "Select Video" and choose `test/fixtures/test-video-10s.mp4`
3. Fill in title, description, and metadata
4. Click "Upload"
5. Monitor progress and verify upload completes

### 3. Test Video Metadata

```bash
# Query the newly uploaded video
curl http://localhost:8787/api/videos \
  -H "Authorization: Bearer YOUR_AUTH_TOKEN" \
  | jq '.data[0]'
```

Expected fields:
```json
{
  "id": "uuid",
  "creatorId": "uuid",
  "cloudflareStreamId": "video-uuid",
  "title": "Test Video",
  "status": "ready",
  "visibility": "public",
  "durationSeconds": 10,
  "viewsCount": 0
}
```

### 4. Test Playback

```bash
# Get signed playback URL
curl http://localhost:8787/api/videos/{videoId} \
  | jq '.playbackUrl'

# Open in browser or VLC
open "https://customer.mux.com/..." # or VLC
```

## Test Video Specifications

### Minimal Test Video (Generated by Script)
- **Duration**: 10 seconds (configurable)
- **Codec**: H.264 (libx264)
- **Audio**: AAC 440Hz sine wave
- **Resolution**: 1280x720
- **Size**: ~500-800 KB
- **Bitrate**: Optimized for fast encoding

### Pre-generated Fixtures

Store test videos in `/test/fixtures/` directory:

```
test/fixtures/
├── test-video-10s.mp4       # 10 sec, ~500 KB (for quick tests)
├── test-video-30s.mp4       # 30 sec, ~1.5 MB (for duration tests)
└── test-video-invalid.txt   # Invalid format (for error handling tests)
```

## Testing Workflows

### Workflow 1: Basic Upload & Retrieval

```typescript
// test/video-upload.test.ts
import { test, expect } from "@playwright/test";

test("should upload and retrieve video metadata", async () => {
  // 1. Get upload URL
  const uploadRes = await fetch("http://localhost:8787/api/videos/upload-url", {
    method: "POST",
    headers: { Authorization: `Bearer ${AUTH_TOKEN}` },
  });
  const { uploadUrl, videoId } = await uploadRes.json();

  // 2. Upload video file
  const videoFile = fs.readFileSync("test/fixtures/test-video-10s.mp4");
  const uploadRes2 = await fetch(uploadUrl, {
    method: "POST",
    body: videoFile,
  });
  expect(uploadRes2.ok).toBe(true);

  // 3. Retrieve video metadata
  await new Promise(r => setTimeout(r, 2000)); // Wait for Stream processing
  const videoRes = await fetch(`http://localhost:8787/api/videos/${videoId}`);
  const video = await videoRes.json();

  expect(video.status).toBe("ready");
  expect(video.durationSeconds).toBeGreaterThan(0);
});
```

### Workflow 2: Streaming & Playback

```typescript
test("should stream and play video", async () => {
  const videoRes = await fetch(`http://localhost:8787/api/videos/${videoId}`);
  const { playbackUrl } = await videoRes.json();

  // Verify playback URL is a valid Cloudflare Stream URL
  expect(playbackUrl).toContain("customer.cloudflarestream.com");

  // Test in Playwright browser
  await page.goto(playbackUrl);
  const player = await page.locator("video");
  await expect(player).toBeVisible();
});
```

### Workflow 3: Upload Rate Limiting

```typescript
test("should apply rate limits to uploads", async () => {
  const results = [];

  // Attempt 5 rapid uploads
  for (let i = 0; i < 5; i++) {
    const res = await fetch("http://localhost:8787/api/videos/upload-url", {
      method: "POST",
      headers: { Authorization: `Bearer ${AUTH_TOKEN}` },
    });
    results.push(res.status);
  }

  // After N requests, should hit 429 Too Many Requests
  expect(results.some(status => status === 429)).toBe(true);
});
```

## Troubleshooting

### Video Upload Fails

**Problem**: 413 Payload Too Large
```
Solution: Check UPLOAD_MAX_SIZE in wrangler.toml
Current: 100 MB
```

**Problem**: 401 Unauthorized
```
Solution: Ensure AUTH_TOKEN is valid and not expired
curl -H "Authorization: Bearer $AUTH_TOKEN" \
  http://localhost:8787/api/auth/session
```

**Problem**: 500 Stream API Error
```
Solution: Verify Cloudflare Stream credentials
- Check STREAM_API_TOKEN in env
- Check STREAM_ACCOUNT_ID is correct
- Test with Stream CLI: npx wrangler stream validate
```

### ffmpeg Installation Issues

**Ubuntu/Debian**:

```bash
sudo apt-get update

ffmpeg -version
```

**macOS**:



## Integration with CI/CD

### GitHub Actions Example

```yaml
name: Video Upload Tests

on: [push]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Install ffmpeg
        run: sudo apt-get install -y ffmpeg
      
      - name: Generate test video
        run: node test/fixtures/generate-test-video.js
      
      - name: Run tests
        run: pnpm test:video
        env:
          WORKER_URL: ${{ secrets.WORKER_URL }}
          AUTH_TOKEN: ${{ secrets.TEST_AUTH_TOKEN }}
```

## Key Endpoints for Video Testing

| Endpoint | Method | Auth | Purpose |
|----------|--------|------|---------|
| `/api/videos/upload-url` | POST | ✅ Required | Get Cloudflare Stream upload URL |
| `/api/videos` | GET | Optional | List all public videos |
| `/api/videos/{id}` | GET | Optional* | Get video details & playback URL |
| `/api/videos/{id}` | PATCH | ✅ Required | Update video metadata |
| `/api/videos` | POST | ✅ Required | Create video record |
| `/api/videos/{id}` | DELETE | ✅ Required | Delete video |

*Optional for public videos; required for private videos
